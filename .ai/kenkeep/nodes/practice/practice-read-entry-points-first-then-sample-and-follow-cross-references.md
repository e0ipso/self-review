Top-level entry points (README, ARCHITECTURE, CONTRIBUTING, docs root) usually frame project vocabulary, name major components, and establish conventions. Read these completely first.

For other docs, sample representative content and follow links. If a README mentions "see docs/architecture/auth.md," that's a high-signal pointer to follow. For large reference docs (e.g. method-by-method API listings), skim section headers and only read prose sections, skipping auto-generated tables.
